## 2. 专项覆盖

1. `0006` 正确承接 `0005`，包含历史大小写冲突预检、`lower(username)` 唯一索引和平台审计表。
2. 内存 fixture 的大小写并发创建仅一个成功；另一请求返回 `duplicate_username`。
3. bootstrap 与普通创建的唯一冲突统一映射为 `duplicate_username`，且只识别指定用户名约束。
4. 登录失败审计保存 HTTP `request_id`。
5. production 对缺失/内存 `RATE_LIMIT_MODE` 启动失败。
6. gateway 缺可信代理 CIDR、Redis 缺 URL 均启动失败。
7. Redis 共享限流契约通过 fake backend 验证。
8. 未受信任 `X-Forwarded-For` 不能改变限流 key。
9. actor 治理和 reserved 扫描工具默认只读、无删除/自动更新语句。
10. PostgreSQL 空库脚本包含两个管理员 Session 并发创建大小写变体用户名的 `201 + 409 duplicate_username` 验收步骤。
