## 8.2 D2 S1/S2 OCR／AI 增量验收

### 8.2.1 基础设施

- 新增 `PaddleOcrAdapter` 并实现既有 `OcrAdapter` Protocol；模型缓存固定在项目内 `apps/api/models/ocr/official_models`。
- 支持 PNG、JPEG 和多页 PDF，返回文字与页码、bbox、confidence 证据；单文件上限 10MB，并验证扩展名、MIME 与 magic bytes 一致性。
- 项目依赖锁定为 `paddlepaddle==3.2.0`、`paddleocr==3.6.0`、`pypdfium2==5.12.1`。
- S1/S2 共用 `ImageScanUpload`，支持拖拽、点击、移动端拍照、预览与图片裁剪；浏览器继续通过 Next.js 同源 API proxy，不新增 HTTP 旁路。
- `request_id` 仅接受首字符为字母或数字、后续为字母数字或 `._:-`、总长度不超过 128；非法输入由服务端生成 UUID 替代。

### 8.2.2 S1 标书／文书

- `POST /api/v1/steven/tenders/{tender_id}/scan`：上传后执行 PaddleOCR，再以正则和关键词匹配截止日期、预算、供应商及条款等字段。
- OCR/规则结果先进入 `review_candidates(needs_review)`；低置信或不确定字段标记待确认，人工确认前不写正式文书数据。
- 确认扫描候选后写入当前草稿，并自动触发既有 `tender_proofreading`；校对仍由 `AiReviewPanel` 逐条接受或忽略。
- `GET /api/v1/steven/tenders/templates/{template_id}/preview` 返回 HTML 预览；三类模板可切换，未替换 `{{...}}` 变量高亮，输出进行 HTML escape，不需先生成 Word。

### 8.2.3 S2 采购比价

- `POST /api/v1/steven/quotes/{quote_id}/scan`：上传后执行 PaddleOCR，再调用 DeepSeek 进行 `quotation_extraction` 严格结构化。
- 输出 Schema 仅允许 `supplier_name`、`quote_date` 和 `items[{item_code,item,specification,qty,unit,unit_price}]`。
- Prompt 与本地 Pydantic 校验共同禁止 `recommend`、`rank`、`approve`、`order`、`total` 与 `select_supplier` 等决策字段。
- 结果只进入 `review_candidates(needs_review)` 并跳转既有 `scan-review`；人工修订和确认前不写正式报价表，Provider 失败或超时进入 failed。
