# Steven Demo 甲方待确认问题清单 v1

- 日期：2026-07-17
- 用途：D1 联通与后续 S1/S2/S3 实施前的最少业务、安全和环境确认

## 1. 采购业务口径

| 待确认事项 | 当前 Demo 处理 | 需要甲方确认 |
|---|---|---|
| 所需报价数量 | `required_quote_count=3`，仅为演示假设 | 不同金额/品类是否要求不同报价数量，缺报价时允许哪些例外 |
| 采购金额门槛 | 未配置 | 各金额区间对应的采购方式、审批层级和额外文件要求 |
| 审批人及层级 | 使用 Steven、审批人、管理员职责分离的技术角色 | 实际岗位、代理审批、多人会签、退回及重新提交规则 |
| 非最低价推荐 | 必须人工填写理由和审批意见 | 是否还需指定理由分类、附件或更高级审批 |
| 报价有效期 | 过期报价显示异常，不自动剔除或推荐 | 过期报价是否允许进入审批，例外批准人是谁 |
| 税费口径 | Demo 按供应商级固定金额 | 学校正式税费、折扣、运费和汇率处理口径 |

## 2. 数据与记录治理

| 待确认事项 | 当前状态 | 需要甲方确认 |
|---|---|---|
| 记录保留期 | `pending_client_confirmation` | 报价原件、OCR/AI 输出、人工修订、审批、审计和导出文件分别保留多久 |
| 删除与归档 | 未实现自动删除 | 谁可批准归档/删除，是否要求法律保留或诉讼保全 |
| 数据分类 | D0 只使用完全脱敏演示资料 | 哪些真实字段允许进入系统，哪些字段必须禁止上传云服务 |
| 审计读取 | 管理员全量、Steven 事项级 | 校方审计、校长/财务/IT 的读取和导出权限 |

## 3. 云 OCR / AI 许可

D1 联网前必须确认：

1. 是否允许将脱敏报价文件发送至 Azure Document Intelligence。
2. 是否允许将 OCR 后的脱敏文本发送至 DeepSeek。
3. 允许的数据地域和 endpoint。
4. Provider 是否承诺不用于模型训练。
5. Provider 的请求日志、内容保留期和删除机制。
6. 是否允许保存 OCR bbox、原文片段和置信度。
7. 哪些字段必须在外发前删除或替换。
8. 网络失败时是否允许使用明确标记的 `cached_demo`，以及缓存有效期。
9. 校方指定的人工复核责任人和复核留痕要求。

当前默认：

- 云 OCR 数据传输关闭。
- AI/OCR 默认关闭。
- Mock/缓存不得标记为实时结果。
- AI/OCR 输出必须人工确认。

## 4. D1 环境批准

D1 开始前需要项目负责人明确批准：

- 启动/连接本机 `postgresql-x64-18` 服务。
- 创建独立数据库 `puiying_steven_demo`。
- 创建独立最低权限应用账号。
- 提供受控 migration/bootstrap 凭据和独立文件根目录。
- 执行 `alembic upgrade head`、脱敏 seed、API 重启恢复与并发 smoke。
- 配置 Azure Document Intelligence 测试 endpoint、模型和受管 Key。
- 配置 DeepSeek 测试 endpoint、模型和受管 Key。
- 允许的测试时间、网络出口、日志脱敏和故障停止条件。

## 5. S1/S3 后续确认

### S1

- 校方正式 Word 模板及启用审批人。
- 模板变量清单和必填规则。
- 标书/书面报价单的审批层级。
- 正式文件命名、签章、保留和修改规则。
- AI 条款草稿允许使用的数据范围。

### S3

- SKU 编码来源和唯一性管理责任人。
- 库位、单位和盘点频率。
- 安全库存和目标库存由谁维护。
- 盘点差异及补货建议由谁确认。
- 是否需要补货审批，以及审批层级。

## 6. 建议最小决策顺序

1. 先确认 D1 是否允许启用独立 PostgreSQL Demo 环境。
2. 再确认云 OCR 数据传输许可和字段白名单。
3. 再确认 DeepSeek 脱敏文本许可和保留政策。
4. 最后确定采购门槛、报价数量、审批层级和记录保留期。

上述事项未确认前，D0 保持 Mock/离线合同状态，不进入真实联通或受控试运行。
