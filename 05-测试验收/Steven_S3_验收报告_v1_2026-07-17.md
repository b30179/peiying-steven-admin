## 1. 实现范围

```text
脱敏库存品项
  → 库位、账面量、安全库存和目标库存维护
  → 发起盘点

  → 已批准 Excel 正式版本化导出
  → PostgreSQL、审计和文件历史恢复
```

实现状态：

- S3 使用既有 Session、CSRF、Origin/Referer、permission-based RBAC、平台审计和 legacy 身份头拒绝。
- S3 使用真实 PostgreSQL Repository 与 Unit of Work；`/health` 显示 `inventory_store=postgresql`，无 InMemory 回退。
- 文件元数据、审计关联和导出版本遵循既有追加式文件存储与可恢复一致性 Saga。
- OCR purpose 仅预留为 `inventory_sheet_extraction`，AI purpose 仅预留为 `inventory_exception_explanation`。
- OCR/AI live 开关保持关闭；候选只能进入通用 `review_candidates(needs_review)`，不得直接写 S3 正式表。
- AI 不确认实物库存、不自动确认补货、不下单、不审批。

## 4. 权限、审计与事务边界

- Steven：读取和维护库存品项、创建/修订盘点、人工确认补货、提交、读取事项级审计及导出已批准版本。
- 审批人：读取、批准或退回；不能任意修改盘点明细。
- 管理员：沿用平台账户、角色、Session 与全量审计职责；默认不自动获得 S3 审批权。
- 同一用户即使兼有 Steven 与审批角色，也不得审批自己提交的盘点单。
- 创建/修订库存品项、创建/修订盘点单、提交、批准/退回及版本预留均由 PostgreSQL 事务保护。
- 审批使用行锁校验状态；状态变化与 S3 业务审计同事务提交。
- 文件发布使用“版本预留 → 临时生成 → 重新打开校验 → hash/size → 追加式发布 → ready 登记”的可恢复 Saga；不宣称数据库与文件系统跨资源原子事务。
- 失败版本号不得回收或复用；历史正式文件不得覆盖或删除。
