# 安全政策

## 报告漏洞

如果您发现安全漏洞，请**不要**提交公开 Issue。

请通过以下方式私下报告：

- 📧 Email: 查看仓库所有者 GitHub 主页

我们会在 48 小时内确认收到，并在 30 天内提供修复方案。

## 安全实践

- 本项目不存储用户敏感数据
- API Key 通过环境变量注入，不硬编码
- RBAC 权限控制防止越权操作
- 所有外部 API 调用通过 HTTPS

## 依赖审计

定期检查依赖安全漏洞。如发现 `requirements.txt` 中的包有已知漏洞，请提交 Issue 或 PR 修复。
