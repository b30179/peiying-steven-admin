# Steven 统一演示脚本 v1

- 日期：2026-07-17
- 目标时长：10–15 分钟
- 唯一浏览器入口：`https://localhost:15443`
- 数据范围：标准脱敏 Demo

## 0. 演示前准备（不计时）

1. 确认 PostgreSQL 既有服务正在运行。
2. 双击 `启动Steven本地Demo.cmd`，或在项目根目录执行：

```powershell
.\scripts\start_steven_demo.ps1
```

3. 确认浏览器打开 `https://localhost:15443/login`，无证书警告。
4. 不在投屏、聊天或文档中展示账号密码、Cookie、Token、私钥或连接串。
5. 审批人登录后若 Dashboard 默认页出现 403，直接使用预先保存的具体事项详情链接；这表示角色职责分离，不是绕过权限。

## 1. HTTPS 安全登录（1 分钟）

1. 打开登录页。
2. 说明浏览器仅访问 loopback HTTPS。
3. 登录后刷新页面，展示 Session 保持。
4. 强调：
   - Session 存储在 PostgreSQL。
   - Cookie 保持 Secure / HttpOnly / SameSite。
   - 写操作受 CSRF 与 Origin/Referer 校验。

## 2. Dashboard 总览（1 分钟）

1. 打开 `/dashboard/steven`。
2. 展示 S1、S2、S3 三个入口。
3. 说明三模块共用：
   - 认证与 RBAC
   - PostgreSQL
   - 审计关联
   - 文件与版本治理原则
   - HTTPS
4. 说明 OCR/AI 仅有离线合同与候选层，尚未联网。

## 3. S2 采购比价（3–4 分钟）

入口：

`https://localhost:15443/dashboard/steven/quotes`

演示顺序：

1. 打开标准脱敏采购事项。
2. 展示 3 家供应商 × 5 个品项。
3. 展示系统总价：
   - SUP-C：2583 HKD
   - SUP-A：2605 HKD
   - SUP-B：2622 HKD
4. 说明运费和税费为供应商级固定金额，只计算一次。
5. 展示系统最低价排序与人工推荐字段分离。
6. 展示非最低价推荐缺理由或审批意见时阻断。
7. 展示已批准事项和 Excel v1/v2/v3 历史。
8. 强调历史文件不覆盖，版本号不复用。

讲解口径：

- 系统负责规则与金额。
- 人工负责推荐与审批。
- AI 不参与供应商推荐、金额、排序或审批。

## 4. S1 标书／文书（3 分钟）

入口：

`https://localhost:15443/dashboard/steven/tenders`

演示顺序：

1. 展示脱敏“采购／服务邀请文书”模板。
2. 展示受控变量、日期、预算和供应商。
3. 演示一个规则阻断：
   - 未替换 `{{...}}`，或
   - 截止日期不足生成日期加 3 天，或
   - 重复供应商。
4. 修正后展示 Word 草稿预览。
5. 展示已批准事项和审批意见。
6. 展示 Word v1/v2/v3 历史。

讲解口径：

- 草稿与正式版严格区分。
- 只有 approved 事项可以正式导出。
- AI 未来只能生成待人工复核的条款候选。

## 5. S3 消耗品库存（3 分钟）

入口：

`https://localhost:15443/dashboard/steven/inventory`

演示顺序：

1. 展示 5 个脱敏库存品项与 2 个演示库位。
2. 展示 SKU 唯一、非负整数、目标库存不低于安全库存规则。
3. 打开标准盘点单。
4. 展示账面量、盘点量、差异、低库存和建议订货量。
5. 展示人工补货量偏离建议且没有理由时阻断。
6. 展示已批准盘点单和 Excel v1/v2/v3 历史。

讲解口径：

- 差异和建议订货量由服务端计算。
- 人工确认补货；AI 不确认实物库存、不自动补货、不下单。

## 6. 审批与自审阻断（1 分钟）

1. 说明 Steven 负责创建、修订、提交。
2. 使用独立审批人查看待审批事项。
3. 强调 S1/S2/S3 均禁止提交人审批自己的事项。
4. 强调退回必须填写意见。
5. 不演示通过修改客户端字段伪造状态、actor 或版本号。

## 7. 审计与重启恢复（1 分钟）

1. 展示事项级审计或版本历史。
2. 说明 request_id、actor、对象 ID 和版本号形成追溯关系。
3. 说明应用重启后，Session、三模块数据、审计和版本元数据从 PostgreSQL 恢复。
4. 说明文件系统与数据库使用可恢复 Saga，不宣称跨资源原子事务。

## 8. 登出与边界声明（1 分钟）

1. 登出。
2. 验证旧 Session 无法继续访问受保护内容。
3. 最终声明：
   - 当前为本机、独立、脱敏 Demo。
   - Azure、DeepSeek、Redis 和外部服务均未连接。
   - D2 未批准。
   - 当前验收不代表生产可用或可受控试运行。

## 9. 现场异常处理

- 页面未打开：先运行停止脚本，再重新启动；不要手工结束来源不明进程。
- 登录后 403：确认当前角色和具体业务页面权限；审批人可直接打开事项详情。
- 证书警告：停止演示并确认当前用户证书信任状态，不允许退回 HTTP 登录。
- 数据被修改：先执行 reset dry-run，核对摘要和 plan hash，再按批准流程执行真实 reset。
- Provider 状态：如果页面显示 OCR/AI，必须明确标注为 Mock/候选；不得称为实时在线结果。
