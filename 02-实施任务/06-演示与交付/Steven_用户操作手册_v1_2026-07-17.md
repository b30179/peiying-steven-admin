# Steven 用户操作手册 v1

- 日期：2026-07-17
- 适用范围：本机独立脱敏 Demo
- 浏览器唯一入口：`https://localhost:15443`

## 1. 启动

前置条件：

- 既有 PostgreSQL 18 服务已运行。
- 既有 `puiying_steven_demo` 数据库和 Demo 应用账号可用。
- 项目 `.venv` 与前端依赖已经准备完成。

启动方式：

- 双击 `启动Steven本地Demo.cmd`；或
- 在 `D:\ZM\AI工坊\某中学\04-开发实现` 执行：

```powershell
.\scripts\start_steven_demo.ps1
```

启动成功后访问：

`https://localhost:15443/login`

不要把 `127.0.0.1:9000` 或 `127.0.0.1:4300` 当作浏览器登录入口；HTTP 不能作为降低 Secure Cookie、CSRF 或 Origin 校验的替代通道。

## 2. 停止

- 双击 `停止Steven本地Demo.cmd`；或
- 执行：

```powershell
.\scripts\stop_steven_demo.ps1
```

停止脚本只停止项目 Caddy、Next.js 与 FastAPI，不修改 PostgreSQL 服务、数据库、账号、ACL、migration、Caddy 证书信任或系统配置。

重复停止会安全返回。若状态文件异常且项目端口仍在监听，脚本会拒绝停止来源不明的进程。

## 3. 角色职责

### Steven

- S1：创建、修订、预览和提交文书；导出已批准 Word。
- S2：查看报价、人工推荐、提交；导出已批准 Excel。
- S3：维护品项、创建盘点、确认补货、提交；导出已批准 Excel。
- 只能读取相应 scoped 审计。
- 不能审批自己提交的事项。

### 审批人

- 查看待审批事项。
- 批准或退回。
- 退回必须填写意见。
- 不能修改提交人的业务明细。
- 不能审批自己提交的事项。

### 管理员

- 负责账户、角色、Session 撤销和平台审计治理。
- 管理员身份不自动绕过 S1/S2/S3 自审限制。

本手册不包含账号密码。账号和凭据由项目负责人通过受控渠道管理。

## 4. S1 标书／文书

入口：

`https://localhost:15443/dashboard/steven/tenders`

正常流程：

1. 选择脱敏模板。
2. 填写标题、编号、事项、日期、预算、地点、供应商和条款。
3. 生成草稿并处理规则错误。
4. 人工修订。
5. 提交。
6. 由独立审批人批准或退回。
7. approved 后生成正式 Word。
8. 在版本历史中查看 v1/v2/v3。

常见阻断：

- 截止日期早于生成日期加 3 天。
- 预算为负或下限高于上限。
- 同一事项供应商重复。
- 存在未替换 `{{...}}`。
- 未批准、被退回或 `draft_error` 事项尝试正式导出。

## 5. S2 采购比价

入口：

`https://localhost:15443/dashboard/steven/quotes`

正常流程：

1. 打开标准脱敏报价事项。
2. 核对 3 家供应商 × 5 个品项。
3. 查看服务端计算的总价和排名。
4. 录入人工推荐。
5. 非最低价推荐时同时填写理由和审批意见。
6. 提交。
7. 由独立审批人处理。
8. approved 后导出 Excel 并查看版本历史。

标准总价：

- SUP-C：2583 HKD
- SUP-A：2605 HKD
- SUP-B：2622 HKD

系统排名不等于人工决策；客户端不能提交可信排名、金额、审批状态或版本号。

## 6. S3 消耗品库存

入口：

`https://localhost:15443/dashboard/steven/inventory`

正常流程：

1. 查看或维护脱敏库存品项。
2. 创建盘点单。
3. 录入盘点量。
4. 查看服务端计算的差异、低库存和建议订货量。
5. 人工确认补货数量。
6. 偏离建议时填写理由。
7. 提交。
8. 由独立审批人处理。
9. approved 后导出 Excel 并查看版本历史。

常见阻断：

- 重复 SKU。
- 数量为负数或非整数。
- 目标库存低于安全库存。
- 同一盘点单重复库存品项。
- 补货量偏离建议但缺少理由。
- draft、submitted 或 returned 状态正式导出。

## 7. 演示数据重置

默认 dry-run：

```powershell
.\.venv\Scripts\python.exe .\scripts\reset_steven_demo_data.py --dry-run
```

`--dry-run` 可省略，默认行为相同；建议保留显式参数作为人工操作记录。

或双击：

`重置Steven演示数据.cmd`

Dry-run 只显示将处理的标准脱敏 Demo 数量和 plan hash，不写数据库、不移动文件。

真实执行前：

1. 停止项目。
2. 保存并复核 dry-run 输出的 plan hash。
3. 确认数据库为 `puiying_steven_demo`。
4. 执行：

```powershell
.\.venv\Scripts\python.exe .\scripts\reset_steven_demo_data.py --apply --confirm-local-redacted-demo --expected-plan-hash <dry-run hash>
```

Reset 只处理固定标准脱敏命名空间，不删除账号、角色、权限、Session schema、migration、非标准 Demo 或非 Demo 数据。

如果 dry-run 显示非预期对象，立即停止，不得添加绕过参数。

## 8. 常见问题

### 启动提示已有实例

先运行停止脚本。若已有实例健康，新的启动命令会直接复用；若实例半启动，脚本会拒绝覆盖。

### 页面返回 403

确认当前角色与页面职责。审批人默认 Dashboard 可能没有 Steven 编辑权限，可直接进入被分配的具体事项详情。

### HTTPS 证书警告

停止操作并联系项目负责人检查当前用户证书信任。不得改用 HTTP 绕过。

### S1/S2/S3 没有预期演示数据

先执行 reset dry-run。只有确认目标全部属于标准脱敏命名空间后，才可按受控参数真实执行。

### OCR/AI 显示失败或不可用

当前 live 开关关闭，这是预期边界。不得注入 Key 或把 Mock 结果称为实时成功。

## 9. 安全禁止事项

- 不输入真实学校、学生、供应商、联系人或业务数据。
- 不展示账号密码、Cookie、Token、私钥或连接串。
- 不关闭 Secure、HttpOnly、SameSite、CSRF、RBAC、Origin/Referer 或审计。
- 不把 Demo 当作生产系统或受控试运行系统。
- 未获 D2.0 明确批准前，不得连接 Azure、DeepSeek、Redis 或其他公网服务。
