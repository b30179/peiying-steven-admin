# Steven 演示数据重置说明 v1

- 日期：2026-07-17
- 适用数据库：`puiying_steven_demo`
- 适用范围：固定标准脱敏 Demo 命名空间
- 默认行为：dry-run

## 1. 目的

将 S1/S2/S3 标准脱敏演示事项恢复为可重复演示状态，同时保留：

- 数据库、schema 和 Alembic revision
- 账号、角色、权限和 Session schema
- 平台审计与既有业务审计历史
- 非标准 Demo 和非 Demo 数据
- 项目配置、Caddy、证书、启停脚本

Reset 不是“清空数据库”，也不是无痕删除。

## 2. 标准命名空间

Reset 只识别固定标准脱敏对象：

- S1 模板与标准文书编号前缀
- S2 标准采购事项名称
- S3 标准盘点单编号与 5 个标准 Demo SKU

所有目标必须同时满足脚本的标准命名空间和 Demo 标记条件。出现非预期对象时脚本应停止。

## 3. 默认 dry-run

先停止项目：

```powershell
.\scripts\stop_steven_demo.ps1
```

执行：

```powershell
.\.venv\Scripts\python.exe .\scripts\reset_steven_demo_data.py --dry-run
```

省略 `--dry-run` 时行为相同；显式参数更适合人工复核和审计记录。

Dry-run 仅输出：

- 数据库名
- Alembic revision
- S1/S2/S3 目标行数
- 文件总数与模块分布
- 明确保留范围
- plan hash

Dry-run 不写数据库、不移动文件、不删除任何对象。

## 4. 真实执行

只有在 dry-run 摘要完全符合预期后，才可执行：

```powershell
.\.venv\Scripts\python.exe .\scripts\reset_steven_demo_data.py --apply --confirm-local-redacted-demo --expected-plan-hash <dry-run hash>
```

三个确认条件缺一不可：

1. `--apply`
2. `--confirm-local-redacted-demo`
3. 与紧邻 dry-run 一致的 `--expected-plan-hash`

如果数据库、revision、行数、文件数或 preserved 范围异常，禁止执行。

## 5. Reset Saga

```text
生成计划与 hash
→ PostgreSQL advisory lock
→ 创建 staging S1/S2/S3
→ 验证状态、金额、版本与文件
→ 写 journal
→ 事务内复核计划
→ 旧标准文件进入 quarantine
→ 原子切换标准命名空间
→ 写 reset 审计
→ post-switch 验证
```

数据库与文件系统之间采用可恢复一致性 Saga，不宣称跨资源原子事务。

## 6. 文件处理

- 旧标准导出文件先移动到：
  - `data\demo-reset-quarantine\<run_id>\`
- 新 seed 文件先在 staging 命名空间生成并验证。
- 历史非 Demo 文件不得删除、覆盖或复用。
- Quarantine 和 journal 仅可在另行批准的清理阶段处理。

## 7. 失败处理

失败时：

- 脚本返回非零退出码。
- 保留 journal。
- 标准数据切换前失败时，原标准数据保持不变。
- 已移动文件按脚本恢复路径处理。
- staging 数据不自动删除，避免隐藏失败或误删。

禁止：

- 手工删除 staging、journal 或 quarantine 来伪装成功。
- 修改脚本跳过 plan hash、数据库、revision 或命名空间校验。
- 在应用运行中强制执行。
