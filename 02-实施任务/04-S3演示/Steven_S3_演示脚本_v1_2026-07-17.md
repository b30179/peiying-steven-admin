# Steven S3 消耗品库存模块演示脚本 v1

- 日期：2026-07-17
- 演示范围：本机独立脱敏 Demo
- 建议时长：5–8 分钟
- HTTPS 入口：`https://localhost:15443`

## 1. 演示前检查

1. 运行 `D:\ZM\AI工坊\某中学\04-开发实现\scripts\start_steven_demo.ps1`。
2. 确认浏览器使用 `https://localhost:15443/login`，不要使用内部 HTTP 端口登录。
3. 确认 `/health` 为：
   - `auth_mode=session`
   - `persistence.mode=postgresql`
   - `inventory_store=postgresql`
   - `demo_seed_enabled=false`
   - `ocr_enabled=false`
   - `ai_enabled=false`
4. 只使用项目负责人保管的受控 Demo 账号；演示材料必须保持脱敏。

## 2. Steven 用户演示

### 步骤 1：HTTPS 登录

- 从 HTTPS 登录页登录。
- 说明 Session 使用 Secure/HttpOnly Cookie，前端不能传入角色或 actor。

### 步骤 2：打开 S3

- 进入 Dashboard。
- 点击“消耗品库存”。
- 页面地址：`https://localhost:15443/dashboard/steven/inventory`。

### 步骤 3：查看或创建脱敏库存品项

- 展示 5 个脱敏品项和两个 `DEMO-STORE-*` 演示库位。
- 新建一个演示品项，填写 SKU、名称、类别、库位、账面量、安全库存和目标库存。
- 强调页面和导出只使用脱敏 Demo 内容。

### 步骤 4：展示库存规则阻断

选择一至两个阻断演示即可，完整规则由自动化报告覆盖：

1. 输入已有 SKU 的大小写或全角等价形式，保存应被重复 SKU 规则阻断。
2. 输入负数或小数数量，保存应被非负整数规则阻断。
3. 将目标库存设为低于安全库存，保存应被阻断。

### 步骤 5：发起盘点

- 新建脱敏盘点单。
- 选择 5 个演示品项，录入本次盘点量。
- 展示系统由服务端计算：
  - 差异数量；
  - 低库存标记；
  - `max(0, 目标库存 - 本次盘点量)` 建议订货量。
- 指出系统不会确认实物库存真实性。

### 步骤 6：人工确认补货

- 选择白板笔案例，将人工确认补货量设置为不同于系统建议。
- 不填写理由时保存应被阻断。
- 填写“脱敏演示：保留两件缓冲量”等人工理由后保存成功。
- 说明系统只记录人工决定，不会自动下单。

### 步骤 7：提交

- 点击提交审批。
- 展示状态变为“待审批”。
- 尝试使用提交人审批，系统应返回 403，证明自审被阻断。

## 3. 审批人演示

### 步骤 8：批准或退回

- 登出 Steven，使用独立审批人账号登录。
- 打开同一盘点单。
- 可先演示“退回但不填写意见”被阻断。
- 若演示退回：填写意见后退回，证明 returned 状态不能正式导出；由 Steven 修订后重新提交。
- 最终由审批人填写人工意见并批准。

审批人不得伪造提交人、版本号或审计 actor；差异和建议订货仍由服务端控制。

## 4. 正式 Excel 与版本历史

### 步骤 9：导出 v1、v2、v3

- 返回具备导出权限的 Steven 账号。
- 对已批准盘点单连续执行三次正式导出。
- 确认生成 v1、v2、v3，文件名和 storage key 不同。
- 分别下载并打开三个 Excel，确认均可读取。

Excel 内容至少包括：

- 盘点单基础信息；
- SKU、品项和库位；
- 账面量、盘点量和差异；
- 安全库存、目标库存和低库存标记；
- 系统建议订货量；
- 人工确认补货量和理由；
- 审批状态、版本号和生成时间。

### 步骤 10：查看历史

- 在详情页查看版本历史。
- 强调：
  - 历史版本不覆盖；
  - 版本号不回收或复用；
  - 文件有 size、SHA-256、创建人和状态元数据；
  - 文件发布与数据库登记采用可恢复 Saga。

## 5. 登出验证

### 步骤 11：退出登录

- 点击登出。
- 刷新受保护页面或使用旧会话访问，预期返回未认证并跳转登录。
- 说明 Session 已在 PostgreSQL 中撤销，不能靠保留旧 Cookie 继续访问。

## 6. 演示讲解边界

必须明确：

- 当前完成的是 S3 本机离线脱敏 Demo。
- 系统计算差异和建议订货量，但不确认实物库存、不自动补货或下单。
- 所有补货数量和审批结论均由人工确认。
- OCR/AI live 开关关闭，未调用 Azure 或 DeepSeek。
- 未来 S3 OCR/AI 只能形成 `needs_review` 候选，不能直接写正式库存或盘点数据。
- S1/S2/S3 三个业务模块均已完成本机脱敏 Demo。
- D2 在线 OCR/AI 尚未批准。
- 本演示不代表生产可用或可受控试运行。
