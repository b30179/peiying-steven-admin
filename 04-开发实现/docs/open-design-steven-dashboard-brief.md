# Open Design Brief：Steven Dashboard（Phase 1）

## 目标与用户

为香港培英中学校务处行政文员 Steven 设计桌面优先的采购行政工作台。页面的首要目标是：Steven 在一个视口内判断优先事项，并以人工复核为前提进入对应业务处理。

## 范围

- 唯一路由：`/dashboard/steven`
- 仅展示 Steven 模块：标书与报价单、采购比价、消耗品库存
- Phase 1 仅使用脱敏 mock 数据；不连接真实 AI、OCR、供应商搜索、邮件或订单系统
- 不呈现其他岗位、管理员入口或跨角色数据

## 信息架构

左侧固定 Steven 专属导航，只含：

1. 总览（当前页）
2. 标书与报价单
3. 采购比价
4. 消耗品库存
5. 历史记录
6. 个人设置

顶部栏显示当前角色“Steven / 采购行政”、通知占位和头像菜单，但不展示其他岗位入口。

## Dashboard 内容

### 页面标题区

- 眉题：`Steven · 採購行政工作台`
- 标题：`今日优先处理`
- 简短说明：`所有建议均须由您复核；系统不会自动审批、选择供应商或下单。`
- 唯一主操作：`新建标书`
- 次操作使用次级按钮或菜单：`新建比价`、`录入盘点`

### 指标卡（四张）

1. 待审批文书：2，琥珀色状态提示
2. 待补报价：3，橙红色异常提示
3. 低库存：1，红色异常提示
4. 本月已导出：8，中性完成状态

### 优先待办（最多五项）

- 礼堂空调保养：书面报价邀请 Draft v2，待上级审核；操作：继续处理
- 2026 暑期清洁用品：缺少 1 家供应商报价；操作：录入报价
- A4 影印纸：低于安全库存，建议订货 20 包；操作：查看库存
- 毕业礼堂布置：截止日期将至；操作：检查资料
- 美术室耗材：等待人工确认盘点差异；操作：查看差异

每项需包含模块图标、状态 Badge、相对时间/截止日及单一操作按钮。

### 最近活动

保留简短三至四条时间线：生成草稿、导入报价、更新盘点、导出版本。提供“查看历史记录”链接。

### 三个模块入口

以清晰的业务卡片进入“标书与报价单”“采购比价”“消耗品库存”，突出各模块的人工复核与当前异常数。

## 交互与状态

- 主界面必须有 loading、空状态与错误状态设计。
- 筛选、导入、导出、审计、附件详情等低频操作使用右侧抽屉或弹窗，不挤占首屏。
- 使用统一状态 Badge、异常色、Toast 和 Confirm Dialog。
- 所有提交、导出和人工确认动作必须是显式操作；明确强调“不会自动审批、不会自动选择供应商、不会自动下单”。
- 右侧复核抽屉展示字段来源、规则/AI 说明（本轮为禁用占位）、异常原因与人工确认入口。

## 视觉与可访问性

- 延续参考后台的“深青绿侧栏 + 浅灰工作区 + 白色卡片”行政工作台气质，但不要复制 Tommy 文件归档业务或布局。
- 风格稳重、清晰、低干扰，适合学校行政场景；中英混合标签应自然。
- 颜色必须同时用文本和图标/Badge 辅助表达状态；保持可读对比度、明确焦点状态和可识别的错误提示。
- 桌面优先，同时提供窄屏折叠导航的响应式策略。

## 禁止项

- 不生成 Apple、Danielle、Tommy、Wendy、梁小姐、Admin 的页面、菜单或数据入口。
- 不虚构已部署、已审批、已导出真实文件或真实供应商数据。
- 不出现自动采购决策、自动审批、自动下单、自动发信、实物库存自动确认。
